# Gemini Code Assist 设置指南

## 概述
本文档说明如何在 1602 Lucky Draw 项目中使用 Gemini Code Assist 进行智能代码辅助。

## 配置文件说明

### 1. 主配置文件 (config.json)
包含项目基本信息、Gemini 模型配置和代码分析设置。

### 2. 项目上下文 (context.md)
详细描述项目结构、技术栈和核心功能，帮助 AI 更好地理解项目。

### 3. 编码规范 (coding-standards.md)
定义项目的代码风格和最佳实践，确保生成的代码符合项目标准。

### 4. API 参考 (api-reference.md)
包含 Firebase、Gemini API 和项目特定 API 的使用示例。

## 使用建议

### 代码生成
当需要 Gemini 生成代码时，请参考以下模式：

1. **Firebase 操作**: 使用 v9 模块化 SDK
2. **样式设计**: 优先使用 TailwindCSS 类
3. **响应式设计**: 包含移动端适配
4. **错误处理**: 总是包含 try-catch 块
5. **多语言**: 支持中英文切换

### 代码审查
Gemini 可以帮助审查代码的：
- 性能优化建议
- 安全性检查
- 代码规范遵循
- 最佳实践应用

### 功能扩展
在添加新功能时，Gemini 会考虑：
- 与现有代码的兼容性
- Firebase 数据结构的一致性
- UI/UX 的连贯性
- 多语言支持

## 常见使用场景

### 1. 添加新的抽奖奖品
```
请帮我在轮盘中添加一个新的奖品类型，包括数据库结构和前端显示
```

### 2. 优化用户体验
```
请分析当前的用户注册流程，并提供改进建议
```

### 3. 添加新的管理功能
```
请在管理面板中添加批量导入用户的功能
```

### 4. 性能优化
```
请检查当前的 Firebase 查询是否可以优化
```

## 注意事项

1. **API 密钥安全**: 不要在代码中硬编码 API 密钥
2. **数据验证**: 总是验证用户输入
3. **错误处理**: 提供友好的错误提示
4. **响应式设计**: 确保在所有设备上正常工作
5. **性能考虑**: 避免不必要的 Firebase 查询

## 更新配置

如果项目结构发生变化，请更新相应的配置文件：
- 新增文件类型时更新 `config.json`
- 技术栈变更时更新 `context.md`
- 编码规范调整时更新 `coding-standards.md`
- API 变更时更新 `api-reference.md`
